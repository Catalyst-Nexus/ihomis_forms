# Hospital Laboratory Upload Page

This app provides a hospital-focused upload page for laboratory result PDFs.

- Request and patient context are loaded from API response data.
- The upload form focuses only on PDF attachments and optional clinical notes.
- Drag-and-drop multi-file upload and per-file retry are supported.

## Environment Setup

Create a .env file in this project directory (ihomis_forms/ihomis_forms) using .env.example as a guide.

Supported variables:

- VITE_LAB_UPLOAD_API_URL (preferred full upload endpoint)
- VITE_API_URL (fallback endpoint if preferred key is not set)
- VITE_LAB_UPLOAD_CONTEXT_URL (optional context endpoint for preloading request/patient data)
- VITE_LAB_UPLOAD_API_TOKEN (optional bearer token)

Optional exact-key mapping variables (comma-separated response paths):

- VITE_LAB_CONTEXT_PANEL_KEYS
- VITE_LAB_CONTEXT_REQUESTED_AT_KEYS
- VITE_LAB_CONTEXT_PATIENT_FIRST_KEYS
- VITE_LAB_CONTEXT_PATIENT_MIDDLE_KEYS
- VITE_LAB_CONTEXT_PATIENT_LAST_KEYS
- VITE_LAB_CONTEXT_PATIENT_FULLNAME_KEYS
- VITE_LAB_CONTEXT_ENCCODE_KEYS
- VITE_LAB_CONTEXT_FHUD_KEYS
- VITE_LAB_CONTEXT_DOCOINTKEY_KEYS

These mapping variables are checked first, then the built-in fallback keys are used.

For the current backend dbController wiring, context can be read from:

- GET /api/db/henctr
- sample row keys: data[0].enccode, data[0].fhud, data[0].docointkey
